Configuración de VAL Telegram

Paso 1: Crear el Bot en Telegram

Abre Telegram y busca al usuario @BotFather (la cuenta oficial con un check azul). Inicia una conversación y envía el comando /newbot.

Sigue los pasos y proporciónale el nombre deseado y un username que termine en "bot" (ej. VAL_DigiKawsay_Bot). Al finalizar, @BotFather te entregará un HTTP API Token similar a este:

1234567890:AAH_XxYyZz1234...

Copia el token real que te otorguen. ¡No lo compartas con nadie!

Paso 2: Configurarlo en el Sistema

Dirígete a la raíz del repositorio de DigiKawsay en tu servidor y abre el archivo .env (si no existe, créalo). Pega tu Token en la variable TELEGRAM_BOT_TOKEN.

TELEGRAM_BOT_TOKEN="Pega_Aqui_El_Token_Real"

A continuación, asegúrate de reiniciar los servicios para que channel-layer tome el nuevo token.

docker-compose restart channel-layer

Paso 3: Establecer el Webhook

Finalmente, debemos enlazar los servidores de Telegram con el channel-layer (ejecutándose en local). Puedes utilizar Ngrok para exponer el puerto 8001 al mundo:

ngrok http 8001

Copia el enlace HTTPS que Ngrok genera. Posteriormente, en otra terminal envía el siguiente request a la API de Telegram con tu URL y tu Token, apuntando al endpoint /webhook:

curl -F "url=https://[TU_NGROK_URL]/webhook" https://api.telegram.org/bot[TELEGRAM_BOT_TOKEN]/setWebhook

Si recibes un {"ok":true,"result":true,"description":"Webhook was set"}, ¡Todo listo! VAL ya puede interactuar con Telegram.
